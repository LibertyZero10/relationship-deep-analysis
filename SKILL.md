# Relationship Deep Analysis - 亲密关系深度心理学分析系统 v2.0

> 从聊天记录的统计数据推导深层心理、关系本质和行为模式。
> 数据基底：微信聊天JSON（sender/content/type/timestamp字段）
> 
> **v2.0 改进**：一键采集脚本 + 月度趋势 + 每千条归一化 + 情感爆发检测 + 跨框架相关矩阵 + 报告模板

## 适用场景

- 用户提供了聊天记录数据（JSON格式，含sender/content/type/timestamp）
- 用户要求对恋爱关系/亲密关系进行深度心理分析
- 用户要求"推导深层东西"、"不只是表面统计"、"心理层面分析"
- 需要从行为数据推导依恋类型、沟通模式、心理防御、关系动力学等

## 使用前提

- 聊天数据已导出为JSON格式（如 wechat-decrypt 导出的 single_xxx.json）
- 每条消息至少包含：sender（发送者标识）、content（文本内容）、timestamp（时间戳）
- 可选字段：type（消息类型：text/call/voice/image/transfer等）
- Python 3.8+（仅标准库，无需额外安装）

## 执行流程（v2.0 三步法）

### Step 1: 一键数据采集

```bash
python scripts/run_analysis.py <json路径> [选项]
```

**选项：**
- `--output, -o` 输出JSON文件路径（默认：`<input>_analysis.json`）
- `--sender-a NAME` 指定A方（用户）标识
- `--sender-b NAME` 指定B方（对方）标识
- `--quiet` 只保存JSON，不输出控制台摘要

**脚本自动完成：**
1. 24个框架全部数据采集（A1-F24）-- 单次遍历，约30秒/10万条消息
2. 月度趋势追踪 —— 所有关键指标按月聚合
3. 每千条消息归一化 —— 消除消息量差异导致的偏差
4. 情感爆发检测 —— 连续≥5条负面情绪消息序列
5. 跨框架相关矩阵 —— 月度时间序列Pearson相关系数
6. 额外指标：回复时间分布、对话分析、时段分析、消息长度分布

**输出：** `<input>_analysis.json`（完整数据 + 控制台摘要）

**自动识别双方：** 脚本通过消息频率自动识别A/B双方。如需手动指定：
```bash
python scripts/run_analysis.py data.json --sender-a "your_id" --sender-b "partner_id"
```

### Step 2: 数据解读与深度推演

读取JSON结果文件，逐框架填充 `templates/report_template.md` 中的占位符。

**每个框架的推演模板：**
1. 展示原始数据 + 归一化数据
2. 计算关键比值（戈特曼比率、蔑视/批评比、BIS:BAS等）
3. 与阈值对照（参考 verified-patterns.md 的公式表）
4. 月度趋势分析（恶化/改善/稳定）
5. 跨框架交叉验证（利用相关矩阵发现强相关对）
6. 写深度推演（从数据到心理动机的推导）

**关键解读原则：**
- **归一化优先**：比较双方时用 per_1k 值，不用绝对值
- **趋势优先**：月度趋势比静态聚合值更有预测力
- **近零频率优先**：B8乔哈里窗中，近零频率话题比高频话题更有诊断价值
- **跨层交叉**：最重要的诊断来自跨框架交叉验证（如A3的Child状态占比高 + D15的高比例压抑 + E20的高惩罚性沉默率 = 三重确认"无Adult在场"）
- **情感爆发加权**：爆发次数和强度比单条消息匹配更能反映真实情感状态

### Step 3: 生成最终报告

用 `write_file` 输出完整报告，按 `templates/report_template.md` 结构组织：
1. 数据概览表
2. A-F六层24框架（每框架含数据表 + 深度推演）
3. 情感爆发分析
4. 跨框架相关矩阵
5. 月度趋势分析
6. 综合诊断（关系本质 + 核心矛盾 + 疗愈方向 + 走向预测）
7. 24框架数据速览附录

## v2.0 改进详解

### 改进1: 一键采集脚本（解决最大痛点）

**v1.0 问题**：每个框架需手写Python脚本塞进 `python -c '...'`，bash单引号嵌套中文正则频繁报EOF，大数据量需遍历多次（每层一次），耗时大。

**v2.0 解决**：`scripts/run_analysis.py` 单次遍历完成全部24框架数据采集。脚本自包含（仅Python标准库），接受JSON路径参数，输出完整JSON结果。

### 改进2: 月度趋势分析（纵向维度）

**v1.0 问题**：所有指标都是全量聚合值。但"蔑视在增加还是减少"比"蔑视总共多少次"重要得多——戈特曼比率恶化趋势是分手的预测因子。

**v2.0 解决**：脚本自动按月聚合关键指标（四骑士月度频次、戈特曼比率、消失次数、分手词频等），输出 `monthly_trends` 字段。报告中展示趋势表 + 解读。

### 改进3: 每千条消息归一化

**v1.0 问题**：双方消息量不同时，直接比绝对频次会高估消息多的一方的"毒性"。

**v2.0 解决**：所有频率指标增加 `per_1k`（每千条消息）归一化值。报告中同时展示绝对值和归一化值。

### 改进4: 情感强度加权（情感爆发检测）

**v1.0 问题**：说一次"生气"和连续30条消息的崩溃，在关键词匹配中都只算1次/条。但后者的情感强度是前者的几十倍。

**v2.0 解决**：脚本检测连续≥5条含负面情绪词的消息序列，标记为"emotional burst"，计算爆发强度 = 消息数 × 平均负面词密度。输出爆发次数、平均强度、Top 5最强爆发事件。

### 改进5: 报告模板自动化

**v1.0 问题**：最终报告完全手写，数据到推演的映射靠人工完成。

**v2.0 解决**：`templates/report_template.md` 用Jinja2风格占位符（`{{ A1.contempt.a }}`、`{{ C12.attachment_type_a }}` 等）。读取JSON结果后填充占位符生成报告骨架，AI只需补充深度推演部分。

### 改进6: 跨框架相关矩阵

**v1.0 问题**：skill提到"跨层交叉验证"但没有系统化方法。

**v2.0 解决**：脚本计算24个框架核心指标的月度时间序列两两Pearson相关系数，自动标出强相关对（|r|>0.5）。AI做综合诊断时有数据支撑。

### 改进7: 额外指标增强

v1.0 只有框架内数据。v2.0 脚本额外采集：
- **回复时间分布**：中位/均值/P25/P75（秒）
- **对话分析**：对话数、平均长度、发起率/结束率
- **时段分析**：早/午/晚/深夜分布、峰值时段
- **消息长度分布**：中位/均值/P75/P95/最大值
- **消息类型分布**：text/call/voice/image/transfer等

## 24框架理论参考

以下为各框架的理论背景，供深度推演时参考。实际数据采集已由 `run_analysis.py` 自动完成，无需手写脚本。

### A层：沟通模式分析

#### A1. 戈特曼四骑士 (Gottman's Four Horsemen)

**理论**：John Gottman通过"爱情实验室"观察数千对伴侣，发现四种沟通模式能以**93%准确率预测关系终结**。越早出现、频率越高，关系越危险。

**四骑士**：
1. **批评 (Criticism)**：攻击对方人格而非行为。"你就是个自私的人" vs "你这件事让我不舒服"
2. **蔑视 (Contempt)**：superiority + 不尊重。嘲讽、翻白眼、侮辱性称呼、阴阳怪气。**这是最强的分手预测因子**
3. **防御 (Defense)**：反击或扮演受害者。"我不是那个意思""你自己不也..."
4. **筑墙 (Stonewalling)**：Withdrawal。沉默、消失、拒绝回应。85%的筑墙者是男性

**关键指标**：蔑视/批评比值（>1.0危险）；戈特曼比率（积极/消极，>5:1健康，<0.8分手概率>90%）

#### A2. 非暴力沟通 (NVC)

**理论**：Marshall Rosenberg的NVC模型。暴力沟通 vs 非暴力沟通的区别在于是否区分观察与评价、感受与判断、需要与策略、请求与命令。

**NVC成熟度** = 非暴力/(暴力+非暴力)×10。>7健康，<5不及格。

#### A3. 交互分析 (Transactional Analysis)

**理论**：Eric Berne的PAC模型。Parent（教导/批评）、Adult（理性/客观）、Child（情感/直觉）。
- 互补交易 = 沟通顺畅
- 交叉交易 = 冲突来源（C→A、A→C等）
- 隐藏交易 = 操控

**关键指标**：交叉交易率 = 交叉/(交叉+互补)，>30%为高冲突。

#### A4. 卡普曼戏剧三角 (Drama Triangle)

**理论**：Stephen Karpman。迫害者、拯救者、受害者三角色循环切换。A指责B→B反击→A道歉→循环。

### B层：心理结构分析

#### B5. 图式治疗 (Schema Therapy)

**理论**：Jeffrey Young的早期适应不良图式(EMS)。18种图式分5大领域：分离与拒绝、自主受损、缺乏界限、他人导向、过度警觉与抑制。

**关键图式碰撞**：一方的"被抛弃"图式触发另一方的"情感压抑"图式 -> 追-逃循环。

#### B6. 心理防御机制 (Defense Mechanisms)

**理论**：Anna Freud & George Vaillant。L1病理性（否认/投射）→ L2不成熟（被动攻击/退缩）→ L3神经症性（合理化/反向形成）→ L4成熟（幽默/升华）。

**陷阱**：中文聊天中"哈哈""呵呵"不一定是真正的成熟防御，更可能是用笑声稀释情绪。报告中应对"幽默"防御加注解。

#### B7. 内在小孩分析 (Inner Child)

**理论**：基于精神分析和创伤理论。退行行为暴露未愈合的童年创伤。补偿寻求暴露童年匮乏。

#### B8. 乔哈里窗 (Johari Window)

**理论**：公开区（双方都知道）、隐藏区（自己知道对方不知道）、盲区（对方知道自己不知道）、未知区（双方都不知道）。

**关键**：近零频率话题比高频话题更有诊断价值。双方都回避的话题最能揭示关系结构。

### C层：关系动力学分析

#### C9. 关系辩证理论 (Relational Dialectics)

**理论**：Leslie Baxter。三对永恒张力：自主vs连接、新奇vs可预测、开放vs封闭。

#### C10. 社会交换理论 (Social Exchange Theory)

**理论**：Thibaut & Kelley。满意度 = 结果 - 比较水平(CL)。承诺度 = 满意度 - 替代比较水平(CLalt) + 投资。

#### C11. 五种权力基础 (Five Bases of Power)

**理论**：French & Raven。奖励/强制/合法/参照/专家权力。

#### C12. 依恋多维模型 (Attachment Model)

**理论**：Bartholomew & Horowitz四象限：安全型、焦虑型（全倾）、回避型、恐惧-回避型。

**最差组合**：焦虑型 × 回避型。一方追触发另一方逃，逃又加剧追。

#### C13. 戈特曼信任公式 (Gottman Trust Formula)

**理论**：信任 = 可预测性 + 可依赖性 + 信心。信心 = 未来词频 vs 分手词频。

### D层：情感与需求分析

#### D14. 自我决定理论 (Self-Determination Theory)

**理论**：Edward Deci & Richard Ryan。三种基本心理需求：自主、胜任、关联。

#### D15. 情绪调节策略 (Emotion Regulation)

**理论**：James Gross。压抑型/表达型/重评型/回避型。双方都高比例压抑 = 最危险组合（无人做情绪稳定器）。

#### D16. 马斯洛需求层次 (Maslow's Hierarchy)

**理论**：生理→安全→归属与爱→尊重→自我实现。卡在哪层反映关系发展阶段。

#### D17. 九型人格关系动态 (Enneagram)

**理论**：9种人格类型在亲密关系中的互动模式。两个3号（成就者）组合 = "高能但高耗"。

### E层：暗黑面与病理分析

#### E18. 煤气灯效应检测 (Gaslighting Detection)

**理论**：Robin Stern三阶段：怀疑期→防御期→抑郁期。注意区分"系统性操控"与"焦虑型记忆模糊"。

#### E19. 创伤绑定深度评估 (Trauma Bond Assessment)

**理论**：Patrick Carnes。间歇强化（不可预测的奖惩）是核心机制——与赌博成瘾的神经机制相同。

#### E20. 沉默对待作为武器 (Silent Treatment as Weapon)

**理论**：区分保护性沉默（短时间+有解释）和惩罚性沉默（长时间+无解释）。后者是情感虐待的一种形式。

#### E21. 关系成瘾/共依赖 (Codependency)

**理论**：Pia Mellody。自我价值依赖对方、过度负责、边界模糊、控制行为、否认需求。

### F层：行为模式分析

#### F22. 习惯回路分析 (Habit Loop)

**理论**：Charles Duhigg。线索→惯例→奖赏。追逐-回避循环和争吵-消失-和好循环是最具破坏性的两个。

#### F23. 行为激活/抑制系统 (BIS/BAS)

**理论**：Jeffrey Gray。BIS高=谨慎/回避；BAS高=冲动/趋近。双方BAS主导=难以在情绪高潮中踩刹车。

#### F24. 关系韧性量表 (Gottman Sound Relationship House)

**理论**：Gottman七层关系房屋。正面视角（积极归因）是最关键层——每行为默认正面还是负面意图。

## 执行中积累的教训与陷阱

### 陷阱1：中文聊天"幽默"防御机制误判（关键）

B6防御机制分析中，"幽默"被归为L4成熟防御。但中文聊天中"哈哈""呵呵"绝大多数时候**不是真正的幽默**，而是：
- 用笑声稀释情绪、回避深度交流
- "呵呵"在中文语境中常带讽刺/蔑视意味
- 真正的幽默是能在痛苦中找到洞察并分享

**修正方法**：在报告中对"幽默"防御加注解，区分"真正的成熟幽默"和"用笑声回避"。如果"哈哈"出现频率极高但"升华"极少，应降级评估为**伪装成熟的回避行为**。

### 陷阱2：交易模式(PAC)的sum()计算Bug

A3交互分析中，统计互补/交叉交易时，嵌套列表推导式 `sum()` 对含 `.get()` 的链式 `+` 操作可能因类型不一致而报 `TypeError`。

**修正方法**：v2.0脚本中已修正——分别获取每个值再相加。如手写脚本，避免在一个表达式里链式 `+` 多个 `.get()` 结果。

### 陷阱3："分手"词频需结合"和好"词频解读

C13信任公式中"分手词频"是重要指标，但在某些关系中"分手"被高频使用但97%最终和好——这说明"分手"已**失去威胁性**，变成了一种情绪化互动模式而非真实意愿。必须同时统计"和好/原谅/不分"词频，计算**分手-复合循环率**。

### 陷阱4：高频率话题不如近零频率话题有诊断价值

B8乔哈里窗分析中，双方都频繁讨论的话题诊断价值有限。真正有诊断价值的是**近零频率话题**--双方都回避的话题。这些话题比高频话题更能揭示关系的结构性空洞。

### 陷阱5：单维度数据不足以定性——必须跨层交叉验证

最重要的诊断结论来自跨层交叉：
- A3(C状态占比高) + D15(高比例压抑) + E20(高惩罚性沉默率) = "没有Adult在场+全员压抑+惩罚性沉默" -> 这是关系无法自我修复的根因
- B5(一方被抛弃图式高频) + C12(同方不信任比值高) + E19(间歇强化高分) = 焦虑型依恋×创伤绑定的完整证据链
- C13(分手词远多于未来词) + B8(未来规划话题近零) + F24(正面视角低分) = 承诺悬空的三重确认

**v2.0增强**：利用脚本输出的 `correlations` 字段自动发现强相关对，辅助跨层交叉验证。

### 陷阱6：Bash单引号嵌套导致EOF错误（v2.0已解决）

在terminal中执行含大量中文正则的Python脚本时，如果脚本中有未转义的单引号，bash会报 `unexpected EOF while looking for matching '''`。

**v2.0解决**：使用 `run_analysis.py` 脚本文件，不再需要在 `python -c '...'` 中嵌入代码。

### 陷阱7：归一化必须用于双方比较（v2.0新增）

双方消息量差异大时，直接比绝对频次会高估消息多的一方的"毒性"。

**v2.0解决**：脚本输出 `normalized` 字段，所有指标同时给出 `per_1k` 归一化值。报告中双方对比必须使用归一化值。

### 陷阱8：静态聚合值无法预测走向——必须看趋势（v2.0新增）

长时间的聚合值告诉你"当前状态"，但月度趋势告诉你"走向哪里"。蔑视从每月2次增加到每月15次，比蔑视总次数更有预测力。

**v2.0解决**：脚本输出 `monthly_trends` 字段。报告中必须展示关键指标的趋势方向（恶化/改善/稳定）。

### 陷阱9：情感强度加权缺失（v2.0新增）

说一次"生气"和连续30条消息的崩溃，在关键词匹配中都只算1次/条。但后者的情感强度是前者的几十倍。

**v2.0解决**：脚本检测连续≥5条含负面情绪词的消息序列（emotional burst），计算爆发强度。报告中展示爆发次数和强度对比。

## 注意事项

1. **伦理边界**：这是基于数据的行为分析，不是临床诊断。所有结论应标注"基于聊天数据的推测"
2. **数据局限**：聊天记录只覆盖文字沟通，不涵盖面对面互动、肢体语言、实际生活事件
3. **输出语言**：跟随用户语言（中文用户用中文输出）
4. **深度优先**：用户明确要求"深度"，不要停留在表面统计，要推导到心理动机和潜意识层面
5. **terminal约束**：Windows环境下terminal可能只返回1行输出，v2.0脚本通过`--quiet`选项避免此问题
6. **分步骤完成**：按用户偏好，不要一次性批量并行多请求，逐层分析
7. **参考文件**：已验证的正则表达式模式见 `references/verified-patterns.md`，指标计算公式表也在其中
8. **v2.0脚本自包含**：`run_analysis.py` 仅依赖Python标准库（json/re/sys/os/argparse/math/statistics/collections），无需安装第三方包
